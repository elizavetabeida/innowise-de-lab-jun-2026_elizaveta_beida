# Предметная область
Предметная область представляет сеть продуктовых магазинов, работающий в нескольких странах: Германия, Франция, Италия, Испания, Польша.
Основными сущностями системы являются товары (products), сотрудники (employees), магазины (shops), клиенты (customers) и продажи (sales).
Вспомогательные сущности, такие как категории (categories), города (cities) и страны (countries), обеспечивают классификацию товаров и географическое расположение магазинов.
Операционный процесс фиксирует факты продажи товаров клиентам через конкретных сотрудников в определенное время.
Каждая транзакция в таблице sales регистрирует количество проданного товара, примененную скидку и итоговую стоимость.
Система также учитывает логистические и качественные характеристики товаров, такие как срок годности (vitality_days) и наличие аллергенов.
Ключевые атрибуты сущности employees: ФИО, дата рождения, пол, id города и id магазина.
Вся структура данных нормализована для обеспечения целостности информации о бизнес-процессах торговой сети.

# Выбор и группировка атрибутов
## Сущность products
- product_id - 
- product_name - название продукта.
- price - цена.
- category_id - id категории товара.
- class - классификатор.
- modify_timestamp - время последнего изменения записи.
- resistant - устойчивость.
- is_allergic - содержит ли продукт аллергены.
- vitality_days - срок годности/хранения в днях.

## Сущность employees
- employee_id - id сотрудника.
- first_name - имя.
- middle_initial - инициал отчества/второго имени.
- last_name - фамилия.
- birth_date - дата рождения.
- gender - пол.
- city_id - id города.
- shop_id - id магазина, к которому прикреплен сотрудник.
- hire_date - дата приема на работу.

## Атомарность
Все вышеперечисленные атрибуты являются атомарными, так как в одной ячейке таблицы не хранится список значений и каждый атрибут описывает ровно одно свойство сущности в конкретный момент времени.
